## 一、产品概述与目标

### 1.1 项目背景

芯动力科技是一家专注于国产AI芯片研发的高科技企业，核心产品为基于自研RPP架构的算力芯片（如RPP-R8）。产品具有低功耗、高算力、全面兼容CUDA生态等优势，主要应用于边缘计算和端侧AI推理场景。

当前公司面临以下业务挑战：
- **POC管理复杂**：客户需要严格的软硬件联合POC验证，测试编译器和RPP库能否无缝迁移原有CUDA算法，流程周期长、环节多
- **技术评估需求强**：客户需要评估功耗（TDP）、封装接口（如M.2加速卡）等技术指标，技术决策链路长
- **渠道管理分散**：涉及大量OEM厂商、方案集成商，合作关系复杂，缺乏统一管理平台
- **销售过程不透明**：从线索到成交的全流程缺乏系统化跟踪，影响销售效率

### 1.2 产品目标

构建一套面向半导体行业的定制化CRM系统，实现：

| 目标维度 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **业务效率** | 缩短POC周期，提升销售转化率 | POC周期缩短30%，转化率提升20% |
| **客户管理** | 实现客户全生命周期管理 | 客户信息完整度≥95% |
| **渠道协同** | 强化OEM/集成商渠道管理 | 渠道合作项目数增长40% |
| **技术支撑** | 规范技术评估流程 | 技术评估通过率提升25% |
| **数据洞察** | 提供销售数据分析和决策支持 | 管理决策效率提升50% |

### 1.3 目标用户

**主要用户群体：**
1. 销售团队（销售代表、销售经理、销售总监）
2. 技术支持团队（FAE、技术工程师、技术总监）
3. 渠道管理团队（渠道经理、合作伙伴经理）
4. 管理层（CEO、销售VP、产品VP）

### 3.3 商机管理模块

#### 3.3.1 商机全流程管理

**功能描述：**  
管理从线索到成交的全流程，支持商机阶段推进、预测分析。

**商机阶段定义：**

| 阶段 | 定义 | 关键动作 | 预计周期 | 成交概率 |
|-----|------|---------|---------|---------|
| **线索验证** | 初步接触，验证需求真实性 | 需求确认、关键人识别 | 1-2周 | 10% |
| **需求分析** | 深入了解客户业务场景和技术需求 | 需求调研、方案初步沟通 | 2-3周 | 20% |
| **技术评估** | 客户进行技术可行性评估 | 提供技术资料、技术交流 | 3-4周 | 40% |
| **POC验证** | 客户进行POC测试 | POC申请、实施、结果确认 | 4-8周 | 60% |
| **商务谈判** | 商务条款谈判 | 报价、合同条款协商 | 2-4周 | 80% |
| **合同签订** | 合同签署与归档 | 合同审批、签署、归档 | 1-2周 | 90% |
| **成交** | 订单确认、交付启动 | 订单创建、交付协调 | - | 100% |

**功能要点：**

| 功能项 | 详细说明 | 优先级 |
|-------|---------|-------|
| 商机创建 | 关联客户、产品、预计金额、预计成交日期 | P0 |
| 阶段推进 | 阶段条件检查、推进记录、回退管理 | P0 |
| 商机看板 | 看板视图展示各阶段商机，支持拖拽 | P0 |
| 商机预测 | 基于历史数据的成交预测、业绩预测 | P1 |
| 竞争对手管理 | 竞争对手信息、竞争策略记录 | P1 |
| 商机协作 | 团队成员协作、任务分配、进度同步 | P1 |

#### 3.3.2 商机预测分析

**功能描述：**  
基于商机阶段、历史转化率、客户等级等因素，预测销售业绩。

**预测模型：**
- 加权预测金额 = 商机金额 × 阶段成交概率
- 季度/年度业绩预测
- 商机转化漏斗分析
- 异常商机预警（长期停滞、阶段异常等）

### 3.4 POC管理模块 ⭐核心模块

#### 3.4.1 POC申请与审批

**功能描述：**  
规范POC申请流程，确保资源合理分配。

**POC申请流程：**

```
销售发起申请 → 销售经理审批 → 技术总监审批 → FAE分配 → POC实施 → 结果确认
```

**申请表单字段：**

| 字段 | 说明 | 必填 |
|-----|------|-----|
| 关联商机 | 选择关联的商机 | 是 |
| 客户名称 | 自动带出 | 是 |
| POC目的 | 验证目标、预期结果 | 是 |
| 应用场景 | 边缘计算/端侧推理/其他 | 是 |
| 技术需求描述 | 详细技术需求 | 是 |
| 算法类型 | 目标检测/图像分割/NLP等 | 是 |
| CUDA兼容性要求 | 是否需要迁移现有CUDA算法 | 是 |
| 功耗要求 | TDP范围（W） | 是 |
| 封装接口要求 | M.2/PCIe/其他 | 是 |
| 预计周期 | 预计POC时长 | 是 |
| 样品需求 | 是否需要样品、数量 | 否 |
| 技术支持需求 | 需要的技术支持类型 | 否 |
| 附件 | 技术文档、需求说明等 | 否 |

**审批规则：**
- 销售经理审批：确认商机真实性、客户优先级
- 技术总监审批：评估技术可行性、资源分配

#### 3.4.2 POC实施跟踪

**功能描述：**  
全程跟踪POC实施进度，确保按时交付。

**POC阶段定义：**

| 阶段 | 关键动作 | 交付物 | 负责人 |
|-----|---------|-------|-------|
| **准备阶段** | 环境准备、样品寄送、资料准备 | POC计划书 | FAE |
| **环境搭建** | 硬件安装、软件部署、环境配置 | 环境配置文档 | FAE |
| **算法迁移** | CUDA算法迁移、RPP库适配、性能优化 | 迁移报告 | FAE |
| **性能测试** | 算力测试、功耗测试、稳定性测试 | 测试报告 | FAE |
| **结果验证** | 客户确认测试结果、问题修复 | 验收报告 | FAE+销售 |
| **总结归档** | POC总结、知识库归档 | POC总结报告 | FAE |

**进度跟踪功能：**
- 阶段进度可视化（甘特图/时间轴）
- 关键节点提醒
- 问题记录与跟踪
- 进度日报/周报

#### 3.4.3 POC结果管理

**功能描述：**  
记录POC结果，支持成功案例沉淀和失败原因分析。

**结果类型：**
- **成功**：客户认可，进入商务谈判
- **部分成功**：部分指标达标，需进一步优化
- **失败**：未达到客户要求，记录失败原因

**结果记录字段：**
- POC结论（成功/部分成功/失败）
- 性能数据（算力、功耗、延迟等）
- 客户反馈
- 成功经验/失败原因
- 后续计划
- 相关文档归档

#### 3.4.4 POC知识库

**功能描述：**  
沉淀POC经验，形成可复用的知识资产。

**知识库内容：**
- POC案例库（成功案例、失败案例）
- 算法迁移指南
- 常见问题FAQ
- 技术文档模板
- 最佳实践

### 3.5 技术评估管理模块 ⭐核心模块

#### 3.5.1 技术需求评估

**功能描述：**  
系统化评估客户技术需求，匹配产品能力。

**评估维度：**

| 评估维度 | 评估内容 | 评估方法 |
|---------|---------|---------|
| **算力需求** | TOPS/W、算力密度 | 需求调研、基准测试 |
| **功耗要求** | TDP范围、散热方案 | 技术规格对比 |
| **封装接口** | M.2/PCIe/自定义 | 产品规格匹配 |
| **算法兼容性** | CUDA算法迁移难度 | 算法分析工具 |
| **软件生态** | 编译器、RPP库、驱动 | 兼容性检查清单 |
| **应用场景** | 边缘计算/端侧推理 | 场景匹配度评估 |

**评估流程：**

```
需求收集 → 技术分析 → 方案匹配 → 可行性评估 → 评估报告
```

#### 3.5.2 功耗与封装评估

**功能描述：**  
针对客户功耗和封装需求进行专项评估。

**功耗评估内容：**
- TDP需求确认
- 散热方案评估
- 功耗优化建议
- 能效比计算

**封装评估内容：**
- 接口类型匹配（M.2/PCIe/BGA等）
- 尺寸规格确认
- 定制封装需求评估
- 供应链可行性

#### 3.5.3 CUDA兼容性评估

**功能描述：**  
评估客户现有CUDA算法迁移到RPP架构的可行性。

**评估内容：**
- CUDA算子使用情况
- RPP库兼容性检查
- 迁移工作量评估
- 性能对比预期

**评估工具集成：**
- CUDA代码分析工具
- RPP兼容性检查工具
- 性能模拟器

#### 3.5.4 技术方案管理

**功能描述：**  
管理技术方案文档，支持方案复用和版本管理。

**方案类型：**
- 标准方案（产品规格、应用场景）
- 定制方案（客户特定需求）
- 竞品对比方案

**方案管理功能：**
- 方案模板库
- 方案版本管理
- 方案审批流程
- 方案效果跟踪

### 3.6 渠道管理模块 ⭐核心模块

#### 3.6.1 渠道伙伴管理

**渠道伙伴类型：**

| **OEM厂商** | 将芯片集成到自有产品中 | 采购+技术支持 |
| **方案集成商** | 为客户提供整体解决方案 | 项目合作+返利 |
| **代理商** | 代理销售芯片产品 | 销售+返利 |
| **技术合作伙伴** | 提供技术支持、联合研发 | 技术合作 |

#### 3.6.3 返利与激励管理

**功能描述：**  
管理渠道返利政策、返利计算与结算。

**返利类型：**
- **采购返利**：基于采购金额的返利
- **项目返利**：基于项目金额的返利
- **目标返利**：达成季度/年度目标的额外返利
- **专项激励**：特定产品、特定时期的激励

**返利规则：**
- 返利比例（按渠道等级）
- 返利计算周期（季度/年度）
- 返利发放条件（付款完成、验收通过等）
- 返利形式（现金/货款抵扣）

**返利计算示例：**

```python
def calculate_rebate(partner_level, purchase_amount, is_target_achieved):
    """
    返利计算
    partner_level: 渠道等级（diamond/gold/silver/bronze）
    purchase_amount: 采购金额
    is_target_achieved: 是否达成目标
    """
    rebate_rates = {
        'diamond': 0.08,  # 8%
        'gold': 0.06,     # 6%
        'silver': 0.04,   # 4%
        'bronze': 0.02    # 2%
    }
    
    base_rebate = purchase_amount * rebate_rates[partner_level]
    
    # 目标达成额外返利
    target_bonus = 0
    if is_target_achieved:
        target_bonus = purchase_amount * 0.02
    
    return base_rebate + target_bonus
```
