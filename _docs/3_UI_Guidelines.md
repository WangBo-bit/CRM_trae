## 一、设计原则

### 1.1 B2B 企业级应用设计理念

芯动力科技CRM系统作为面向半导体行业的B2B企业级应用，遵循以下核心设计理念：

| 设计理念 | 说明 | 实践要点 |
|---------|------|---------|
| **高效性** | 提升用户工作效率，减少操作步骤 | 快捷操作、批量处理、智能默认值 |
| **专业性** | 体现专业、严谨的企业形象 | 规范化布局、精准的数据展示 |
| **一致性** | 统一的视觉语言和交互模式 | 统一组件、统一色彩、统一交互 |
| **可扩展性** | 支持业务功能扩展 | 模块化设计、灵活布局 |
| **易用性** | 降低学习成本，快速上手 | 清晰导航、操作引导、错误提示 |
| **数据驱动** | 突出数据价值，辅助决策 | 数据可视化、关键指标突出 |

### 1.2 半导体行业特色设计风格

结合半导体行业特点，融入以下设计元素：

| 特色元素 | 设计体现 |
|---------|---------|
| **科技感** | 深色主题、渐变色、微动效、数据可视化 |
| **专业感** | 规范化布局、精准数据展示、技术参数清晰 |
| **创新性** | 现代化UI风格、流畅交互体验 |
| **可靠性** | 稳定的视觉层次、清晰的状态反馈 |
| **国际化** | 支持中英文切换、国际化设计规范 |

## 二、色彩系统

### 2.1 主色调

主色调体现半导体行业的科技感和专业性，采用深蓝色系作为品牌主色。

| 色彩名称 | HEX值 | RGB值 | 使用场景 |
|---------|-------|-------|---------|
| **品牌主色** | `#1E40AF` | rgb(30, 64, 175) | 主按钮、导航高亮、重要元素 |
| **主色浅色** | `#3B82F6` | rgb(59, 130, 246) | 悬停状态、链接、次要按钮 |
| **主色更浅** | `#60A5FA` | rgb(96, 165, 250) | 禁用状态、背景装饰 |
| **主色极浅** | `#DBEAFE` | rgb(219, 234, 254) | 背景色、标签背景 |
| **主色深色** | `#1E3A8A` | rgb(30, 58, 138) | 按下状态、强调元素 |

**主色渐变：**
```css
/* 主色渐变 - 用于重要按钮、卡片头部等 */
background: linear-gradient(135deg, #1E40AF 0%, #3B82F6 100%);

/* 科技感渐变 - 用于数据可视化、装饰元素 */
background: linear-gradient(135deg, #1E40AF 0%, #6366F1 50%, #8B5CF6 100%);
```

## 三、字体规范

### 3.1 字体家族

#### 3.1.1 中文字体

| 字体类型 | 字体名称 | 使用场景 |
|---------|---------|---------|
| **主字体** | 苹方-简 (PingFang SC) | macOS、iOS 系统默认 |
| **备选字体** | 微软雅黑 (Microsoft YaHei) | Windows 系统默认 |
| **备选字体** | 思源黑体 (Source Han Sans) | 开源字体，跨平台 |
| **兜底字体** | sans-serif | 系统默认无衬线字体 |

#### 3.1.2 英文/数字字体

| 字体类型 | 字体名称 | 使用场景 |
|---------|---------|---------|
| **主字体** | Inter | 现代无衬线字体，适合界面 |
| **备选字体** | SF Pro Display | Apple 系统字体 |
| **备选字体** | Segoe UI | Windows 系统字体 |
| **等宽字体** | JetBrains Mono | 代码、技术参数 |
| **兜底字体** | sans-serif | 系统默认无衬线字体 |

#### 3.1.3 CSS 字体定义

```css
:root {
  /* 中文字体栈 */
  --font-family-chinese: "PingFang SC", "Microsoft YaHei", "Source Han Sans", sans-serif;
  
  /* 英文/数字字体栈 */
  --font-family-english: "Inter", "SF Pro Display", "Segoe UI", sans-serif;
  
  /* 等宽字体栈 */
  --font-family-mono: "JetBrains Mono", "SF Mono", "Consolas", monospace;
  
  /* 完整字体栈 */
  --font-family-base: var(--font-family-english), var(--font-family-chinese);
}
```

| **H4** | 18px | 1.33 (24px) | 表格标题、列表标题 | - |
| **H5** | 16px | 1.5 (24px) | 小标题、标签 | - |
| **正文大** | 16px | 1.5 (24px) | 重要正文、强调文字 | large |
| **正文** | 14px | 1.5 (21px) | 正文、表单标签、按钮 | default |
| **正文小** | 13px | 1.54 (20px) | 辅助文字、说明文字 | small |
| **辅助文字** | 12px | 1.5 (18px) | 提示文字、标签、时间戳 | - |

### 3.3 字重规范

| 字重值 | 字重名称 | 使用场景 |

## 五、组件规范

基于 Element Plus 组件库，定义以下组件的使用规范。

### 5.1 按钮规范

#### 5.1.1 按钮类型

| 按钮类型 | 样式 | 使用场景 | Tailwind 类名示例 |
|---------|------|---------|------------------|
| **主按钮** | 品牌主色背景，白色文字 | 主要操作、提交、确认 | `bg-primary text-white hover:bg-primary-dark` |
| **次按钮** | 白色背景，主色边框和文字 | 次要操作、取消、返回 | `bg-white text-primary border-primary hover:bg-primary-lightest` |
| **成功按钮** | 成功色背景，白色文字 | 成功操作、通过审批 | `bg-success text-white hover:bg-green-600` |
| **警告按钮** | 警告色背景，白色文字 | 警告操作、重置 | `bg-warning text-white hover:bg-yellow-600` |
| **危险按钮** | 错误色背景，白色文字 | 危险操作、删除 | `bg-error text-white hover:bg-red-600` |
| **文字按钮** | 无背景，主色文字 | 辅助操作、链接跳转 | `text-primary hover:text-primary-light` |

#### 5.1.2 按钮尺寸

| 尺寸 | 高度 | 内间距 | 字号 | 使用场景 |
|-----|------|--------|------|---------|

| default | 36px | 10px 20px | 14px | 默认按钮 |
| small | 32px | 8px 16px | 14px | 表格操作、卡片内按钮 |

#### 5.1.3 按钮状态

| 状态 | 样式变化 | 说明 |
|-----|---------|------|
| **默认** | 正常样式 | 默认显示状态 |

## 九、交互动效规范

### 9.1 动效原则

| 原则 | 说明 | 实践要点 |
|-----|------|---------|
| **功能性** | 动效服务于功能，提升用户体验 | 状态变化、操作反馈 |
| **简洁性** | 动效简洁流畅，不干扰用户 | 时长适中，不过度 |
| **一致性** | 动效风格统一，符合品牌调性 | 统一缓动函数、时长 |
| **性能优先** | 动效不影响系统性能 | 使用 CSS 动画、避免重排 |

### 9.2 动效时长

| 动效类型 | 时长 | 使用场景 |
|---------|------|---------|
| **微交互** | 100ms - 200ms | 按钮悬停、图标切换 |
| **过渡动画** | 200ms - 300ms | 弹窗打开、抽屉展开 |
| **页面切换** | 300ms - 500ms | 页面切换、路由跳转 |
| **复杂动画** | 500ms - 800ms | 数据加载、图表动画 |

### 9.3 缓动函数

| 缓动函数 | CSS | 使用场景 |
|---------|-----|---------|
| **线性** | `linear` | 匀速动画，较少使用 |
| **缓入** | `ease-in` | 元素消失、淡出 |
| **缓出** | `ease-out` | 元素出现、淡入 |
| **缓入缓出** | `ease-in-out` | 状态切换、过渡动画 |
| **自定义** | `cubic-bezier(0.4, 0, 0.2, 1)` | Material Design 缓动 |

### 9.4 常用动效示例

#### 9.4.1 按钮悬停动效

}

## 十一、无障碍设计规范

### 11.1 无障碍原则

遵循 WCAG 2.1 无障碍标准，确保系统对所有用户友好。

| 原则 | 说明 | 实践要点 |
|-----|------|---------|
| **可感知** | 信息和UI组件可被用户感知 | 足够的对比度、替代文本 |
| **可操作** | UI组件和导航可被操作 | 键盘可访问、足够的目标尺寸 |
| **可理解** | 信息和UI操作可被理解 | 清晰的标签、错误提示 |
| **健壮性** | 内容可被各种用户代理解析 | 语义化HTML、ARIA标签 |

### 11.2 色彩对比度

| 场景 | 最小对比度 | 推荐对比度 |
|-----|-----------|-----------|
| **正文文字** | 4.5:1 | 7:1 |
| **大标题（≥18px 或 14px 加粗）** | 3:1 | 4.5:1 |
| **图标、边框** | 3:1 | 4.5:1 |
| **非活动UI组件** | 无要求 | - |

### 11.3 键盘可访问性

| 要求 | 说明 |
|-----|------|
| **Tab 导航** | 所有交互元素可通过 Tab 键访问 |
| **焦点可见** | 焦点状态清晰可见 |
| **快捷键** | 提供常用操作快捷键 |
| **跳过导航** | 提供"跳到主内容"链接 |

### 11.4 ARIA 标签

| ARIA 属性 | 使用场景 | 示例 |
|----------|---------|------|
| `aria-label` | 无文字标签的元素 | `<button aria-label="关闭">×</button>` |
| `aria-labelledby` | 关联标签 | `<div aria-labelledby="title">` |
| `aria-describedby` | 关联描述 | `<input aria-describedby="hint">` |
| `aria-hidden` | 隐藏装饰元素 | `<span aria-hidden="true">` |
| `role` | 定义角色 | `<div role="dialog">` |

### 11.5 语义化 HTML

```html
<!-- 语义化页面结构 -->
<header>
  <nav aria-label="主导航">
    <!-- 导航内容 -->
  </nav>
</header>

<main>
  <article>
    <h1>页面标题</h1>
    <section aria-labelledby="section-title">
      <h2 id="section-title">章节标题</h2>
      <!-- 内容 -->
    </section>
  </article>
</main>

<aside>
  <!-- 侧边栏内容 -->
</aside>

<footer>
  <!-- 页脚内容 -->
</footer>
```
